Welcome to HPAHelper’s documentation!¶

A light-weight, shared collection of Python libraries/modules. meant to encapsulate a lot of very commonly used functionality (abstraction of queries to RETS, common heuristics/formulas), providing a universal standard to prevent code duplication.

When creating, or modifying libraries, make sure you document the change in dependencies, Python version! Use the latest Python version (currently 3.x).

Libraries should be separated in their own folders, and the scope of each one should be well defined. Instructions on how to use them is also important, including cross-language use, would be preferable.

All libraries should use the Sphinx Docstring format

Please continue to document any changes or updates to this repo in confluence

You can run unit tests manually by running pytest in terminal. If you get an error of the form No module named 'hpa_helper', run export PYTHONPATH=. in the root directory of the project and try running the tests with pytest again.
